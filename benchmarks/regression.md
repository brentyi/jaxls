# Regression report: jaxls main vs this PR

Generated by `benchmarks/make_regression_report.py` from the JSONs in
`benchmarks/results/regression/` and `benchmarks/results/`. All runs:
one warmup solve (absorbs compilation), then timed jit-cached solves.
Library defaults unless stated; notebook/example rows use each
problem's own solver settings, exactly as a user would run them.

Known benign deltas: the two cart-pole notebooks compare
non-converged snapshots (every config hits the iteration cap on a
non-convex problem; gaps shrink to basin-level noise at higher
caps), and near-threshold iteration counts can differ across
devices (e.g. cam_design terminates at 2 iterations on CPU under
both versions; GPU float reductions land its final cost-delta just
above the 1e-6 tolerance).

## Bundle adjustment (the headline feature)

jaxls@main has no variable elimination: its best options are full-
system CG or cholmod. The PR adds Schur+dense and Schur+CG.
